REPLY KEYBOARD BUTTONS IN GROUPS
════════════════════════════════════════════════════════════════════════════════

⚡ SWITCHED FROM INLINE TO REPLY KEYBOARDS:

Why? Inline buttons are unreliable. ReplyKeyboard buttons:
  ✅ More reliable (always work)
  ✅ Can be dismissed with X button
  ✅ Don't take permanent space
  ✅ Better mobile UX

GROUP BUTTONS:
  🔗 Join Game      ▶️ Start Game      🛑 End Game
  📜 Show Characters 🎯 Sus Points     ✕ Close

Each button also works as a slash command (/command) for extra redundancy.

HOW IT WORKS NOW
════════════════════════════════════════════════════════════════════════════════

GROUP CHAT:
  User taps button → message_handler routes it → game logic executes
  (No callbacks, no unreliable inline buttons)

PRIVATE DM:
  User taps button → message_handler OR callback handler
  (Callbacks still used for inline buttons in DM flows)

FALLBACK:
  User types /command → any command handler
  Works everywhere, always
